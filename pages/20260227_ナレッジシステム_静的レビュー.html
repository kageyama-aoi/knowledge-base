ナレッジシステム 静的レビューレポート

対象：MD→HTML変換パイプライン・index生成・UI/UX・パフォーマンス 方針：現状の問題を優先度別に整理し、改善の方向性を示す

システム全体像（現状）

pages/*.md をpush
    ↓
GitHub Actions 起動
    ↓
pandoc で .md → .html 変換（--standalone のみ）
    ↓
元の .md を pages/old/ に退避
    ↓
build_index.py で index.html を自動生成
    ↓
git commit & push

🔴 バグ（確実に直すべき）

1. NEWバッジが全件に付く

場所： tools/build_index.py is_new() 関数

問題：

def is_new(path: Path) -> bool:
    mtime = datetime.fromtimestamp(path.stat().st_mtime)
    return datetime.now() - mtime <= timedelta(days=7)

CI（GitHub Actions）でcheckoutすると、全ファイルのmtimeがその時点の時刻になる。 結果、毎回全件が「7日以内に更新」と判定されて 全ファイルにNEWバッジが付く。

修正方針： - ファイル名の日付プレフィックス（20260227_）をパースして新旧を判定する - またはgitのコミット日時をAPIで取得する（複雑になるため前者推奨）

2. テーマ切り替えボタンが存在しない

場所： knowledge-ui.js 15行目 / index.html

問題：

const themeBtn = document.getElementById('toggleTheme'); // → 常にnull

index.html に id="toggleTheme" の要素が存在しない。 ダーク/ライト切り替え機能が完全に死んでいる。

修正方針： index.html にテーマ切り替えボタンを追加する、または build_index.py 側で生成する。

🟠 品質問題（早めに対処すべき）

3. 変換後のHTMLにCSSが適用されない

場所： .github/workflows/build-index.yml 29行目

問題：

pandoc "$f" -o "pages/${base}.html" --standalone

--standalone はpandocのデフォルトテンプレートでHTMLを生成するが、knowledge-style.css は一切適用されない。 変換後の個別ページは index.html と全く異なる見た目になる。

修正方針：

pandoc "$f" \
  -o "pages/${base}.html" \
  --standalone \
  --css="../knowledge-style.css" \
  --template=pages/template.html  # カスタムテンプレートを用意

または、pandocテンプレートに <link rel="stylesheet"> と「←一覧に戻る」ボタンを埋め込む。

4. 個別ページから一覧への戻りリンクがない

問題： pandocが生成したHTMLには index.html へのナビゲーションがない。 ページを開いたら戻る手段がブラウザの「戻る」ボタンだけ。

修正方針： pandocのカスタムテンプレートに固定ヘッダーを追加する。

<header>
  <a href="index.html">← 一覧に戻る</a>
</header>

5. デスクトップで「すべて開く/閉じる」ボタンが非表示

場所： knowledge-style.css 243行目・343行目

問題：

.search-actions {
  display: none; /* デスクトップでは非表示 */
}
@media (max-width: 640px) {
  .search-actions {
    display: flex; /* モバイルのみ表示 */
  }
}

ドキュメント数が多い場合、デスクトップでも開閉コントロールは必要。

修正方針： display: none を display: flex に変更し、常時表示にする。

🟡 UX改善（体験を高める）

6. 最新セクションとアーカイブでアイテムが重複表示される

問題： 「🆕 最新の資料（10件）」に表示されるファイルが、月別アーカイブにも同じく表示される。 ユーザーが同じリンクを2回目にする。

修正方針案： - 最新セクションを廃止して月別のみにする - または最新セクションは「本日追加」など期間を絞る - または最新セクションを月別アーカイブと重複しないよう除外する

7. 検索がタイトルのみで本文を検索できない

問題： data-search に含まれるのはタイトルとファイル名のみ。 ドキュメント本文の内容では検索できない。

修正方針（軽量案）： build_index.py で各HTMLの本文テキストを抽出し、data-search に含める（200〜400文字程度）。 修正方針（本格案）： search-index.json を別途生成し、クライアントサイドで全文検索する。

8. 初期表示で全セクションが展開される

場所： build_index.py 154行目、index.html の <details ... open>

問題： 月別アーカイブのすべてのセクションが open 状態で生成される。 ドキュメントが50件以上ある場合、初回ロード時のDOMが大きく、スクロール量も多い。

修正方針： - 直近1〜2ヶ月のみ open、それより古いものは open なしで生成する - build_index.py で日付比較して制御する

9. Escapeキーで検索クリアができない

場所： knowledge-ui.js

問題： / キーで検索フォーカスは実装済みだが、Escape で閉じる処理がない。 キーボードユーザーが検索後にリストに戻れない。

修正方針：

if (e.key === 'Escape' && document.activeElement === searchInput) {
  searchInput.value = '';
  applySearch();
  searchInput.blur();
}

10. 検索入力にデバウンスがない

場所： knowledge-ui.js 184行目

問題：

searchInput.addEventListener('input', applySearch);

入力のたびに全DOMを走査する。ドキュメントが増えると体感遅延が生じる。

修正方針：

let timer;
searchInput.addEventListener('input', () => {
  clearTimeout(timer);
  timer = setTimeout(applySearch, 150);
});

🔵 将来的な改善候補

11. タグ・カテゴリによるフィルタ

現状は日付順のみ。学習ログ・実装手順・リファレンスなどカテゴリ分類があると探しやすくなる。 Issueテンプレートの学習カテゴリと連動させると整合性が取れる。

12. pages/old/ の扱いの明示

変換後のMDが pages/old/ に退避されるが、これがどういう扱いか（参照可能か・削除OKか）がルールとして明文化されていない。 old/ ディレクトリはgit管理の観点から整理ポリシーを決めておく方が良い。
