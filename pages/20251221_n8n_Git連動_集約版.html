全体像

通信経路（必ず頭に入れる）:

GitHub（外） → ngrokの公開URL（外） → localhost:5678（内：n8n）
登場人物 役割
n8n（ローカル） あなたのPCで待ち受ける（http://localhost:5678）
ngrok（中継） 外部公開URLを作り、localhostへ転送する
GitHub（送信側） GitHubサーバーからWebhook URLへPOSTする

重要: GitHubのWebhookに localhost を入れると、GitHubサーバーから見た localhost になりあなたのPCには届かない。

レベル定義

レベル ゴール
レベル1 GitHub push → ngrok → n8n Webhookに届き、OUTPUTに headers/body が表示される
レベル2 mainへのpushのみ絞り込み、Slack/Email/Teamsに「判断できる通知」を送れる
レベル3〜5 複数commit集約、署名検証、本番環境化など（詳細は元の手順書参照）

レベル1：Webhook疎通

Step 1：n8n側 — Webhookノードを作る

設定（学習フェーズ）:

項目 値
HTTP Method POST
Path github-webhook（任意）
Authentication None
Respond Immediately（自動で200 OK）
  • Test URL: Listen for test event 中に受信するためのURL
  • Production URL: ワークフロー有効化後に常時受けるURL
  • 学習フェーズは Test URLで疎通が安全

Step 2：GitHub側 — Webhookを登録する

場所: 対象リポジトリ → Settings → Webhooks → Add webhook

入力値（最小）:

項目 値
Payload URL ngrokのURL + n8nのPath
Content type application/json
Secret 空でOK（後で署名検証するなら設定）
Events Just the push event

Step 2.5：ngrokで公開URLを作る

# 初回のみ：authtokenを登録
ngrok config add-authtoken YOUR_TOKEN

# 起動
ngrok http 5678

Forwardingに表示されたURL（例: https://xxxx.ngrok-free.dev）を確認。

GitHubに貼るPayload URL:

https://xxxx.ngrok-free.dev/webhook-test/github-webhook
  • n8n側のURLは変えない
  • GitHubに貼る先だけをngrokにする

切り分け用: http://127.0.0.1:4040（ngrok管理画面）でPOSTが届いているか確認。

Step 3：動作確認

  1. n8n：Webhookノードで Listen for test event
  2. GitHub：pushを発生させる
# 最短：空コミット
git commit --allow-empty -m "test webhook"
git push

# または：GitHub画面でREADMEを1文字編集してコミット

成功の証拠: - OUTPUTに「1 item」 - headers.user-agent に GitHub-Hookshot - x-github-event: push - body.ref: refs/heads/main

届かない時の切り分け

  1. GitHub: Webhookの Recent Deliveries が 200 か
  2. ngrok: http://127.0.0.1:4040 にPOSTが来ているか
  3. n8n: Listen for test event を押して待機しているか
  4. URL: Payload URL が「ngrokドメイン + n8nのPath」になっているか

レベル2：通知に意味を持たせる

Step 4：IF（mainだけ通す）

Value 1:   {{ $json.body.ref }}
Operation: Equals
Value 2:   refs/heads/main
  • main ではなく refs/heads/main がGitHubの仕様
  • main へのpush → True Branch / 別ブランチ → False Branch

Step 5：Set（通知用DTOを作る）

設定: - Mode: Manual Mapping - Include Other Input Fields: OFF（巨大JSONを捨てる）

作るフィールドと Expression:

// repository（例：owner/repo）
{{ $json.body.repository.full_name }}

// actor（push実行者）
{{ $json.body.pusher.name }}

// branch（refs/heads/ を除去）
{{ $json.body.ref.replace('refs/heads/', '') }}

// commit_message（先頭コミット）
{{ $json.body.commits[0].message }}

// changed_files（added/modified/removedをインデント付きで）
{{
  [
    ...$json.body.commits[0].added,
    ...$json.body.commits[0].modified,
    ...$json.body.commits[0].removed
  ]
  .map(file => `  - ${file}`)
  .join('\n')
}}

// compare_url（差分ページへのリンク）
{{ $json.body.compare }}

Setの出力（合格ライン）:

{
  "repository": "owner/repo",
  "actor": "name",
  "branch": "main",
  "commit_message": "message ...",
  "changed_files": "  - file1\n  - file2",
  "compare_url": "https://github.com/..."
}
  • changed_files と compare_url は別物: ファイル名は commits[].added/modified/removed から取り出す

Step 6：Slackに送る（推奨）

Slackノード設定:

項目 値
Resource Message
Operation Post
Channel #github-notify など

メッセージ本文（スマホで読みやすい）:

🟢 GitHub Push

📁 {{ $json.repository }}
👤 {{ $json.actor }}
🌿 {{ $json.branch }}

📝 {{ $json.commit_message }}

📦 Changed files
{{ $json.changed_files }}

🔗 {{ $json.compare_url }}

設計の意図: 文字整形はSetで済ませ、Slackノードは「埋め込んで送るだけ」にする。後でEmail/Teamsに増やしても同じDTOを使い回せる。

Step 7：Emailに送る（同じDTOを使い回し）

件名: [GitHub Push] {{ $json.repository }} ({{ $json.branch }}) - {{ $json.actor }}

本文:
Repository: {{ $json.repository }}
Actor: {{ $json.actor }}
Branch: {{ $json.branch }}

Message:
{{ $json.commit_message }}

Changed files:
{{ $json.changed_files }}

Compare:
{{ $json.compare_url }}

詰まりポイントまとめ

Q 原因 解決
GitHub Webhookって「どこ」のこと？ リポジトリ単位の設定 対象リポジトリ → Settings → Webhooks
localhost がダメと言われる GitHubは外部サーバーから送る ngrokで公開URLを作る
ngrokってDLが必要？ ローカルで常時接続するクライアントが必要 exe実行（authtoken登録も必要）
ERR_NGROK_4018 authtoken未登録 サインアップ → token取得 → ngrok config add-authtoken
n8n側のURLを変える必要ある？ URLが2種類見えて混乱 n8nは固定。GitHubに貼る先だけngrokにする
GitHub側で何をする？ イベントが必要（待つだけでは来ない） pushを発生させる（空コミットなど）
Setのやり方が分からない 空の表から自分で作る方式 Add Field → name/value → Expressionで埋める
compare_url は資材名？ リンクとファイル名は別物 ファイル名は commits[].added/modified/removed から取る

  • レベル4: added/modified/removed の分類表示
  • レベル5: 署名検証（Secret）＋本番URL化
