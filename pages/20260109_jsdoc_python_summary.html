JSDoc / Python docstring 活用まとめ

〜設計情報からHTML・図・自動化まで〜

1. 結論サマリ

  • JSDocは単なるコメントではなく「設計メタデータ」
  • Webアプリ / Node.js / GAS すべてで同じ価値を持つ
  • Pythonには同等〜それ以上に成熟した仕組みが標準で存在する
  • 公式ツールを使えば HTML生成までは手頃に自動化可能
  • JSDoc / docstring は AI・設計レビュー・引き継ぎに直結する

2. なぜJSDocは「発展できる情報群」なのか

JSDocには、最初から以下の情報が含まれている：

  • クラス・関数名
  • 引数（名前・型・意味）
  • 戻り値
  • 責務（description）
  • 依存関係（型参照）

これらはそのまま：

  • HTMLドキュメント
  • クラス図（UML / Mermaid）
  • 設計レビュー
  • AI入力用メタデータ

に変換できる。

3. Webアプリ・GASでJSDocは使えるか？

結論

完全に使える。しかも価値は同じ。

理由

  • JSDocは実行に影響しない
  • JavaScriptの実行環境に依存しない
  • IDE / ツール / AI が読む
観点 Webアプリ GAS
JavaScriptとして動く ○ ○
JSDocが書ける ○ ○
補完・設計理解 ○ ◎
引き継ぎ資料代わり △ ◎

6. HTML生成 手順（公式ツール）

JavaScript

npm install -D jsdoc
npx jsdoc -c jsdoc.json

Python

pip install sphinx
sphinx-quickstart
sphinx-build -b html source docs

7. GASでもHTML生成できるか？

Node.js側でJSDocを実行すれば可能。

npx jsdoc src/gas -d docs
