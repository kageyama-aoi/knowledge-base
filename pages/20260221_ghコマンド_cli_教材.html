01

重要：コマンドには「前提状態」がある

GH CLIのコマンドは、対象データが存在しないと動かないものがあるというのが最初の重要な気づきです。まず試すなら gh repo view --web から。リポジトリさえあれば前提なしで動きます。

コマンド 必要な前提状態 状態
gh repo view --web リポジトリがあればOK 即使える
gh issue list / create リポジトリがあればOK 即使える
gh pr list / create ブランチがpushされていること 要push
gh run list / view Actionsが最低1回実行済みであること 要実行
gh workflow run .github/workflows/にymlファイルがあること 要yml
02

基本：認証・状態確認

リポジトリがあればOK
bash 
gh auth status   # 認証アカウントの確認・トークン期限切れチェック

現在どのアカウントで認証しているか確認します。トークン期限切れなど問題の第一チェックポイントです。

03

--web フラグ ★ 地味に神コマンド

コマンドに --web を付けると、該当のGitHubページをブラウザで直接開きます。ブラウザでリポジトリを探してメニューをたどる手間が完全になくなります。

★ まず最初に試すべき1コマンド 前提状態が不要で、リポジトリさえあれば今すぐ動く。gh repo view --web でGitHubのリポジトリページが直接開きます。
bash 
gh repo view --web          # ★前提不要。リポジトリのトップページを開く
gh issue view 42 --web     # Issue #42のページを開く
gh pr view 15 --web        # PR #15のページを開く
gh run view --web           # Actions実行ページを開く（要：Actions実行済み）
gh workflow view --web      # ワークフローのページを開く（要：ymlファイル）

ターミナルで作業しながらブラウザも確認したいとき、コンテキストスイッチのコストがほぼゼロになります。

04

Issue操作 ★★★

リポジトリがあればOK（即使える）

一覧を見る

bash 
gh issue list                          # openのIssue一覧
gh issue list --state all             # クローズ済みも含めて全件
gh issue list --label "bug"           # ラベルで絞り込み

Issueを作る

bash 
gh issue create --title "バグ修正" --body "詳細をここに"
gh issue create --title "タイトル" --body "本文" --label "bug" --assignee "@me"
gh issue create --title "タイトル" --body-file issue_body.md   # ★AIとの連携で超重要
★ --body-file がAI連携の核心 AIが生成した内容をMarkdownファイルに書き出してから --body-file で渡すと、自動でIssueを量産できます。

Issueを見る・操作する

bash 
gh issue view 42                       # ターミナルで詳細表示
gh issue view 42 --web                 # ブラウザで開く
gh issue close 42                      # クローズ
gh issue comment 42 --body "対応完了"  # コメント追加

★ ぜひ知っておいてほしいコマンド

bash 
gh issue develop 42 --checkout        # Issue#42紐付きブランチを作ってチェックアウト

Issue番号がブランチ名に含まれるので、GitHubがIssueとPRを自動で関連付けてくれます。Issue→開発フローを自動化する際の核心です。

05

PR操作 ★★★

ブランチがpushされていること

PRを作る

bash 
gh pr create --title "feature: ログイン追加" --body "Close #42"   # マージ時にIssue自動クローズ
gh pr create --draft                                              # ドラフトPR
gh pr create --base main --head feature/login                   # ブランチを明示

一覧・確認・操作

bash 
gh pr list                             # オープンなPR一覧
gh pr view 15                          # PR #15の詳細
gh pr view 15 --web                   # ブラウザで開く
gh pr checkout 15                     # ローカルにチェックアウト（他人のPR確認に便利）
gh pr merge 15 --squash              # スカッシュマージ
gh pr merge 15 --merge               # 通常マージ
06

GitHub Actions連携 ★★★

.github/workflows/ にymlファイルがあり、1回以上実行済みであること

実行状態を確認する

bash 
gh run list                            # 実行一覧（成功・失敗・進行中）
gh run list --limit 5                 # 直近5件
gh run view                            # 最新runの詳細
gh run view --log                      # ログ全体を出力（CIが落ちた原因調査）
gh run view --web                      # ブラウザで確認
gh run watch                           # ★リアルタイム監視（自動化スクリプトでも使える）

ワークフローを手動起動・管理

ymlにworkflow_dispatchトリガーが設定されていること
bash 
gh workflow run deploy.yml
gh workflow run deploy.yml --field environment=staging   # インプット値を渡す
gh workflow list                       # 一覧と有効/無効状態
gh workflow enable deploy.yml
gh workflow disable deploy.yml

リポジトリ操作（補足）

bash 
gh repo view                           # 現在のリポジトリの基本情報
gh repo view --web                    # ブラウザで開く（前提不要）
gh repo clone ユーザー名/リポジトリ名   # クローン
gh repo create my-new-repo --public   # 新リポジトリを作成
08

自動化の入り口：組み合わせパターン

パターン① コミット後にIssueを自動作成（AIと連携）
bash 
# AIが生成したIssue本文をファイルに書き出した後
gh issue create --title "$(cat issue_title.txt)" --body-file issue_body.md --label "auto-generated"
パターン② PRを作ってCIの完了を待つ
bash 
gh pr create --title "fix: バグ修正" --body "Close #42"
gh run watch   # CIが終わるまでここで待機
パターン③ Issue → ブランチ → PR を一気に進める
bash 
gh issue develop 42 --checkout   # Issue#42用ブランチを作ってチェックアウト
# ... 開発作業 ...
git add . && git commit -m "fix: #42対応"
git push
gh pr create --title "fix: #42対応" --body "Close #42"

まとめ：優先的に覚えるコマンド TOP 11

優先度 コマンド 前提状態 できること
★★★ gh repo view --web 即使える 最初に試すべき1コマンド
★★★ gh issue create 即使える IssueをCLIから作成（自動化の核心）
★★★ gh pr create 要push PRをCLIから作成
★★★ gh run list 要実行 Actionsの実行状態確認
★★☆ gh issue list 即使える Issue一覧確認
★★☆ gh pr list 要push PR一覧確認
★★☆ gh run watch 要実行 CI完了まで待機・監視
★★☆ gh workflow run 要yml ワークフロー手動起動
★☆☆ gh pr checkout 要push 他人のPRをローカルで確認
★☆☆ gh issue develop Issue存在 Issue連携ブランチ作成
★☆☆ gh run view --log 要実行 CIのログ確認
