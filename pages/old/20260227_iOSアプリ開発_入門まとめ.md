# iOS アプリ開発 入門まとめ

> Web開発経験者が「自分用iPhoneアプリ」を作るための全体像とつまずきポイントを先に掴む

---

## 前提・ゴール

| 項目 | 内容 |
|---|---|
| 目的 | 自分用のiOSアプリを作り、自分のiPhoneに入れて使える状態にする |
| 配布方法 | **Aルート（完全自分用）** — App Store不要、審査なし |
| 環境 | Mac あり |
| 背景 | Webアプリ中心の経験あり。iOS特有のUI・署名・配布の仕組みが未経験 |

## iOS UI の用語整理

### ドラムロール入力の正式名称

よく「ドラムロールみたいに回して選ぶやつ」と表現されるUIの正式名称：

| 呼び方 | 使う場面 |
|---|---|
| **ホイールピッカー（Wheel Picker）** | UI呼称として最もわかりやすい |
| **ピッカー（Picker）** | 設計・開発現場での総称 |
| スピナー（Spinner） | Android文脈で使われる（iOS文脈では使わない） |

### 実装での名称（これを検索・実装に使う）

**UIKit：**
- `UIPickerView` — 汎用選択ピッカー
- `UIDatePicker` — 日付・時間専用

**SwiftUI：**
- `Picker` + `.pickerStyle(.wheel)`
- `DatePicker` + `.datePickerStyle(.wheel)`

> 「ドラムロール」はイメージとして正しいが、実装・検索には通じない。
> **実装に直結する正式名称に翻訳して使う**ことが重要。

## iOSアプリ開発の全体像（Aルート：自分用）

```
① 開発環境準備（Xcodeの導入）
       ↓
② アプリ実装（最小の1画面から）
       ↓
③ 実機テスト（自分のiPhoneに直接インストール）
       ↓
④ 使いながら改善
```

- App Store公開・審査は**不要**
- Apple Developer Program（年額登録）も最初は**不要**
- 無料アカウントで実機インストールまで完結できる

---

## VS Code と Xcode の役割の違い

### 結論

| ツール | できること | できないこと |
|---|---|---|
| VS Code | コードの編集 | ビルド・署名・実機転送 |
| Xcode | コード編集 ＋ ビルド ＋ 署名 ＋ 実機インストール ＋ デバッグ | — |

**整理：** 編集はVS Codeでもできるが、**動かす最後の責任はXcodeが持つ**。

### なぜXcodeが必須なのか

- iOSアプリは**署名（code signing）がないと端末で動かない**
- iOS SDKとビルドツールはApple提供でXcodeに統合されている
- Webの「保存 → 即ブラウザ確認」とは違い、必ず「ビルド → 署名 → 実行」が入る

---

## Xcodeとは何か（Web開発者視点）

Web開発の感覚に置き換えると：

> **VS Code ＋ ビルド環境 ＋ Apple署名システム ＋ 実行環境** が一体化したもの

### 画面構成

```
┌─────────────────────────────────────────┐
│  上：実行（▶︎） ／ 実機・シミュレータ選択    │
├──────┬──────────────────────┬───────────┤
│ 左    │       中央            │    右     │
│ナビ   │    エディタ（コード）   │インスペクタ│
│ゲータ  │  SwiftUI Preview     │  設定・属性│
│ファイル│                      │          │
├──────┴──────────────────────┴───────────┤
│  下：デバッグ（エラー・ログ）               │
└─────────────────────────────────────────┘
```

**最初に使う領域はこの3つだけで十分：**

- 左：プロジェクトファイル
- 中央：SwiftUIコード
- 上：▶︎実行ボタン

## ゼロからiPhoneに入れるまでの時間見積もり

**合計目安：2.5〜4時間**（回線速度・Mac性能による）

| 工程 | 目安時間 |
|---|---|
| Xcodeダウンロード（約15GB） | 30〜90分（最長工程） |
| 初回起動＋追加コンポーネント | 10〜15分 |
| 新規プロジェクト作成 | 10〜20分 |
| 実機接続＋署名設定 | 20分前後 |
| ビルド→実機インストール | 10〜20分 |

---

## よくつまずくポイント（技術ではなく"Appleの仕組み"）

### つまずきTop3

1. **署名（Signing）周り** — 設定が通らない・エラーが出る
2. **iPhone側の「開発者を信頼」設定** — これをしないとアプリが開けない
3. **Apple IDログイン・2段階認証** — 認証タイミングで詰まりやすい

### 無料アカウントの制限（知っておくと焦らない）

> **インストールしたアプリは7日で期限切れになる**

- 期限が切れてもアプリのデータは消えない
- 再ビルドしてインストールし直せば復活する
- 自分用Aルートでは許容範囲内

## 実行計画：2日間で「実機インストール」まで

### Day 1（金曜夜）：環境構築

**ゴール：Xcodeが起動してApple IDが登録済みの状態**

事前確認：

- [ ] Macの空き容量が **30GB以上** あるか
- [ ] macOSが最新（または推奨バージョン）か
- [ ] 回線が安定しているか

手順：

- [ ] App StoreからXcodeをインストール（約15GB・最大90分）
- [ ] 初回起動（追加コンポーネントのインストール・規約同意）
- [ ] Xcode Settings → Accounts → Apple IDを追加
- [ ] 起動OK・エラーなし・Apple IDが表示されているか確認

---

### Day 2（土曜）：初めてのアプリ実行

**ゴール：SwiftUIテンプレートアプリをシミュレータ・実機で動かす**

- [ ] 新規プロジェクトを作成する（SwiftUI選択）
- [ ] シミュレータで ▶︎ 実行する
- [ ] 余力があれば実機に接続して「開発者を信頼」設定まで
- [ ] 実機インストールを確認する

---

## その後の進め方（実機インストール後）

自分用アプリのテーマに合わせて：

1. **画面設計** — 1画面で完結する最小構成から始める
2. **データ設計** — まず永続化なし → 必要になったら `UserDefaults` / `SwiftData`
3. **使いながら改善** — 動いてからが本番
