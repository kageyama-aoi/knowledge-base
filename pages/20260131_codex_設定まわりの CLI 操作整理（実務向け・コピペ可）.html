Codex 設定まわりの CLI 操作整理（実務向け・コピペ可）

① 概念：Codex 設定ディレクトリ（.codex）

  • 粒度：概念／設計
  • レイヤー：アプリ（Codex）
  • 役割：Codex が設定・プロンプトを探索する既定ディレクトリ
  • 補足：
    • 実行対象ではない
    • OS や PATH とは無関係

② コマンド：ディレクトリ作成（PowerShell）

  • 粒度：コマンド解説
  • レイヤー：PowerShell
  • 目的：Codex 用プロンプト格納ディレクトリを作成
mkdir $HOME\.codex\prompts -Force

③ コマンド：ファイル作成（PowerShell）

  • 粒度：コマンド解説
  • レイヤー：PowerShell
  • 目的：Codex に読ませるプロンプト定義ファイルを作成
New-Item $HOME\.codex\prompts\greet.md -ItemType File

④ 概念：PATH（環境変数）

  • 粒度：概念／OS仕様
  • レイヤー：OS
  • 役割：
    • 実行ファイル（exe / cmd / bat）の探索パス定義

PATH 確認（PowerShell）

$env:PATH

PATH を 1 行ずつ確認

$env:PATH -split ";"

⑤ コマンド：VS Code 起動（PowerShell）

  • 粒度：コマンド解説
  • レイヤー：PowerShell → OS
code

⑥ コマンド：特定ファイルを VS Code で開く

code $HOME\.codex\prompts\greet.md -

⑦ 概念：Get-Command（PowerShell のコマンド解決）

  • 粒度：概念／調査手段
  • レイヤー：PowerShell
  • 役割：
    • PowerShell がコマンド名をどう解釈しているかを確認

コマンド解決を確認

Get-Command code

実体パスのみ確認

Get-Command code | Select-Object Source

⑧ 概念比較：PATH と Get-Command の違い

観点 PATH Get-Command
レイヤー OS PowerShell
主用途 実行ファイル探索 コマンド解決確認
対象 exe / cmd / bat 上記＋PS固有要素
用途例 実行可否 原因調査・確認

⑤ Downloadsですぐ開く

① ワンライナーで「名前を確認するだけ」 Get-ChildItem “$HOME” -File -Filter *.md | Sort-Object LastWriteTime -Descending | Select-Object -First 1 Name, LastWriteTime

② ワンライナーで「確認してから開く（表示＋実行）」 Get-ChildItem “$HOME” -File -Filter *.md | Sort-Object LastWriteTime -Descending | Select-Object -First 1 | Tee-Object -Variable latest | Select-Object Name, LastWriteTime; code $latest.FullName

最終要約

  • 設定ファイル作成と実行環境設定は別レイヤー
  • PATH は OS、Get-Command は PowerShell の仕組み
  • 粒度を意識すると CLI 手順が構造として読める
